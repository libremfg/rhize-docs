## Breaking Changes

### "Save Version As" Mutation
The return of a "Save Version As" mutation no longer returns a complete version of the copied entity. Instead, it will return a shallow copy, which excludes results for the `versions` field.

#### Example

With the following mutation for Material Definitions:

```graphql
mutation {
  saveMaterialDefinitionVersionAs(
    fromID: "ExampleID"
    fromVersion: "1"
    toVersion: "2"
  ) {
    iid
    version
    versionStatus
    materialDefinition {
      iid
      id
      label
      nextVersion
      versions {
        iid
      }
      activeVersion {
        iid
      }
    }
  }
}

```

The result in the prior version would be:

```json
{
  "data": {
    "saveMaterialDefinitionVersionAs": [
      {
        "iid": "0x7536",
        "version": "2",
        "versionStatus": "DRAFT",
        "materialDefinition": {
          "iid": "0x7531",
          "id": "ExampleID",
          "label": "ExampleID",
          "nextVersion": "3",
          "versions": [
            {
              "iid": "0x7"
            },
            {
              "iid": "0x8"
            }
          ],
          "activeVersion": null
        }
      }
    ]
  }
}
```


The new result would be:

```json
{
  "data": {
    "saveMaterialDefinitionVersionAs": [
      {
        "iid": "0x7536",
        "version": "2",
        "versionStatus": "DRAFT",
        "materialDefinition": {
          "iid": "0x7531",
          "id": "ExampleID",
          "label": "ExampleID",
          "nextVersion": "3",
          "versions": [],
          "activeVersion": null
        }
      }
    ]
  }
}
```
