Create objects from the UI

To make a production object visible to the Rhize data hub, you must define it as a data model. Along with is API, Rhize also has a graphical interface to create and update objects in your role-based equipment hierarchy.

Often, one object references another: for example, a piece of equipment may belong to an equipment class, have a unit of measure as a property, and be associated with process segment. These associations form nodes and edges in your knowledge graph, so the more information relationships that you accurately create, the better.

Procedure

  1. From the UI, select the menu in the top corner.
  2. Select Master Data, then the object you want to configure.
  3. Create new (or the plus sign).
  4. Name the object according to your naming conventions.
  5. Fill in the other fields. For details, refer to the Master definitions and fields.

You can create many different objects, all with their own parameters and associations. For that reason, a general procedure such as the preceding lacks any interesting detail.

To make the action more concrete, the next section provides an example to create plausible group of objects.

Example: create an oven class with an instance

AG holdings is a fictional enterprise that makes product called Alleman Brownies. These brownies are produced in its UK site, AG_House, specifically in the brownie_kitchen_1 work center of the south_wing area.

The brownie_kitchen_1 kitchen has oven_123, an instance of the Oven class. This equipment item also has a data source that gives temperature readings, which are published to a dashboard.

Here’s how you could use the Rhize UI to model this.

Note If you are actively following to learn, make sure to use names that will easily identify the objects as example objects for testing.

Model the equipment levels:

  1. From Master Data, select Equipment and enter AG_house as the name.

  2. Give it a description. Then for Equipment Level, choose Site.

  3. From the new AG_House object, create a sub-object with the + button.

  4. Name the object south_wing and choose Area as its level.

  5. Repeat the preceding steps to make brownie_kitchen1 a work center in the south_wing.

    Once complete, the hierarchy should look like this:

Model the Oven equipment class:

  1. From Master Data, select Equipment Class.
  2. Give it a name that makes sense for your organization. Give it a description, such as Oven for baking.
  3. Add any additional properties.
  4. Create .
  5. Make it active by changing its version state.

Create the associated data source:

  1. From Master Data, select Data Source.
  2. Add the source’s connection string and protocol, along with any credentials (to configure authentication, refer to Agent configuration).
  3. Select the Topics tab and add the label and data type.
  4. Create and make the version active.

Now, create an instance of the Oven.

  1. From Master Data, select Equipment. Then create a sub-object for the brownie_kitchen1 work center.

  2. Add its unique, globally identifiable ID and give it a description.

  3. For Equipment Class, add the Oven class you just created.

  4. For Equipment Level, select WorkUnit.

  5. Create.

    After the object is successfully created, you can add the data source.

  6. From the Data Sources tab, select Link Data Sources. Select the data source you just created.

On success, your UI should now have an item equipment that is associated with an equipment level, equipment class, and data source. For a complete reference of all objects and properties that you can add through the UI, refer to the Master definitions and Fields.
